# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Native Android **multimodal** camera app called **Recon**. The Gradle `namespace` and `applicationId` remain `com.example.bundlecam` so existing installs don't orphan on upgrade; every class, log tag, document, resource, and EXIF identifier is **Recon**. Core flow: capture photos / videos / voice notes (all interleaved in one queue) → stage on internal storage → swipe-commit a "bundle" → background worker copies raw files into per-modality subfolders + produces one vertically-stitched JPEG from the photos into the user's SAF-picked folder. Committed bundles can be transferred peer-to-peer over the LAN via the [LocalSend](https://github.com/localsend/protocol) v2.1 protocol — multi-select bundles in the in-app browser, pick a peer on the same Wi-Fi.

The user chooses the capture modality via a top-bar `VIDEO · PHOTO · VOICE` pill (photo is the hub), or by swiping horizontally on the viewfinder (same carousel). The shutter and control row are modality-aware: photo taps, video/voice tap-to-start / tap-to-stop with a morphing red dot→stop-square fill. Stitching remains photo-only — videos and voice pass through the worker untouched except for SAF copy.

The platform-neutral product + architecture spec (source of truth for an iOS port) lives in `recon-mvp-designs.md`. README.md is the Android-specific technical reference — keep both in sync when changing pipeline, capture-flow, UX, or LocalSend behavior. Post-MVP ideas (OCR companion, background-record FGS) live in `BACKLOG.md`.

Toolchain: JDK 11, Android `compileSdk`/`targetSdk` 36, `minSdk` 26, Gradle 9 via wrapper, Kotlin 2.2.10, Compose BoM 2026.02.01, **CameraX 1.6.0** (`core`/`camera2`/`lifecycle`/`view`/`extensions`/`video`). 1.6.0 unlocked concurrent `Preview + ImageCapture + VideoCapture<Recorder>` binding and switched MP4 writes to Media3 Muxer, which produces playable files even on mid-record process kill (this is load-bearing for OrphanRecovery's video restoration path). No `apply plugin` style — everything is via `libs.versions.toml` version catalog.

## Architecture: the three-phase capture lifecycle

This is the load-bearing mental model. **Don't break the phase boundaries** — especially the Main-thread commit pivot.

**Phase 1 — Shutter (varies, UI-thread budget).** `CaptureViewModel.onShutter` dispatches on `_modality.value`:

- **PHOTO** (≤100ms): CameraX `takePicture()` returns JPEG → `StagingStore.writePhoto` to `filesDir/staging/session-{uuid}/{photo-uuid}.jpg` (filenames are random UUIDs — ordering lives in the manifest AND in the session's `.order` append-log, not the filesystem) → `StagingStore.appendOrderEntry` → `ExifWriter.stamp` (time/GPS-if-cached/orientation) → `decodeThumbnail` → push `StagedItem.Photo` onto `uiState.queue`.
- **VIDEO** (two taps): first tap → `CaptureController.startVideoRecording(filesDir/…/{uuid}.mp4)` (CameraX `Recorder.prepareRecording(...).start()`, `withAudioEnabled` omitted so video is silent and the mic stays free for VOICE). Order-log entry appended at start. Second tap → `stopVideoRecording()` awaits `VideoRecordEvent.Finalize` → extracts poster frame via `MediaMetadataRetriever` → push `StagedItem.Video`.
- **VOICE** (two taps): first tap → `VoiceController.startRecording` via `MediaRecorder` (AAC-LC .m4a 48kHz mono 96kbps). Permission pre-check triggers the `RECORD_AUDIO` launcher if missing. Order-log entry at start. Second tap → `stopRecording` → push `StagedItem.Voice` with duration from `SystemClock.elapsedRealtime` deltas. An amplitude-polling coroutine on `Dispatchers.Default` reads `MediaRecorder.getMaxAmplitude()` at ~30Hz into `amplitudeFlow` for the overlay's waveform.

No SAF writes in Phase 1 for any modality. `BusyState` tracks {Idle, Capturing, Recording, Rebinding}; the sync flip to `Capturing` before launching the async coroutine is the race-prevention gate for rapid double-taps.

**Phase 2 — Commit (single frame).** `CaptureViewModel.onCommitBundle` pivots UI **synchronously on Main**: nulls `currentSession`, clears `queue` + `dividers`. Shutter is usable on the very next frame. The actual bookkeeping (bundle-ID allocation from DataStore, `PendingBundle` manifest serialization + file write, `WorkScheduler.enqueue`) runs in a background coroutine afterwards. The manifest's `orderedItems: List<PendingItem>` is a sealed polymorphic list (`PendingPhoto` / `PendingVideo` / `PendingVoice` — version-tagged `version: Int = 2`). If the process dies between the UI pivot and the manifest save, the staging session persists on disk and `OrphanRecovery` restores it as an uncommitted queue on next launch.

**Phase 3 — Worker (seconds–tens of seconds, off UI).** `BundleWorker` (CoroutineWorker) loads the manifest, runs `BundleWorker.planRouting(manifest)` (a pure companion function that partitions `orderedItems` into photos / videos / voices with 1-based **global capture-order indices**), then dispatches:

- Photos (if `saveIndividualPhotos` and list non-empty): bounded 2s GPS refresh → `stampFinalMetadata` (`UserComment=Recon:{bundleId}:pNNN` + GPS backfill) in one open/save pass → SAF copy into `bundles/{bundle-id}/photos/`.
- Videos (always when list non-empty): SAF copy into `bundles/{bundle-id}/videos/` (no EXIF — MP4 rotation lives in the `tkhd.matrix` set by `Recorder` at start-time; UserComment doesn't apply).
- Voices (always when list non-empty): SAF copy into `bundles/{bundle-id}/audio/` (no EXIF).
- Stitch (if `saveStitchedImage` and photos non-empty): `Stitcher` into `stitched/{bundle-id}-stitch.jpg`. **Photo-only, forever** — video frames are never pulled into the stitch.

Then deletes staging + manifest. **A process-wide `Mutex` (`workMutex`) serializes all workers** — stitching allocates ~60% of heap and two in parallel reliably OOMs mid-range devices. Video SAF-copy is I/O-bound and runs under the same mutex (conservative; perf-split deferred).

### Why these shapes

- **Manifest file indirection (not WorkManager `Data`).** A 50-item bundle's paths exceed WorkManager's 10KB Data limit. Only `bundleId` passes through `Data`; the worker loads `PendingBundle` JSON from `filesDir/pending/{bundleId}.json`. Orphan recovery replays manifests on next launch.
- **Manifest is polymorphic with an explicit version field.** `ManifestStore.load` branches on `version`: v1 hand-decodes the legacy `orderedPhotos: List<PendingPhoto>` into `orderedItems`; v2 uses the standard kotlinx-serialization polymorphic path with `classDiscriminator = "type"` and `@SerialName`s (`photo` / `video` / `voice`). `encodeDefaults = true` so the version field is actually written on encode. Unknown future versions fail loudly (`null` return + log).
- **Staging on internal storage, not in memory.** 50 photos = ~2.5MB of thumbnails, not ~250MB of decoded JPEG. Photos are durable from the moment the shutter fires. Videos stream to disk via CameraX `Recorder`; voice streams via `MediaRecorder` — neither is buffered in RAM.
- **`.order` append-log per session is the source of truth for restored-queue ordering.** Photo `lastModified` ≈ capture time, but video/voice `lastModified` = *stop()* time — sorting by mtime alone would scramble a `[V, P, A]` capture as `[P, A, V]` on restore. The log is appended at *start-of-recording* for video/voice (before the file even exists), and at write-time for photos. `OrphanRecovery` prefers the log when present, falls back to mtime for pre-Phase-D sessions.
- **Bundle ID allocated atomically from DataStore before any SAF I/O.** If the worker fails partway, the counter does not rewind.
- **`WorkScheduler.observeFailures()` filters pre-existing failures** on first emission (see the `acknowledged` set). WorkManager replays historical `WorkInfo`, so without this, every app launch re-surfaces yesterday's transient failure.

## Resilience: `OrphanRecovery`

Runs at `AppContainer` init. Prunes terminal `WorkInfo`; re-enqueues manifests without in-flight work; returns the most-recent orphan staging session (a queue the user was building when the process died) to `CaptureViewModel.init` so it comes back as an uncommitted queue. Older orphan staging sessions are deleted to prevent unbounded disk growth.

Widens the file filter to `.jpg` / `.mp4` / `.m4a`, skips dot-files (`.order`, `.discarded`) and zero-byte files, validates mp4/m4a playability via `MediaMetadataRetriever.extractMetadata(DURATION)` — torn files return null duration and are deleted. Because CameraX 1.6's Media3 Muxer periodically flushes a valid `moov` atom during recording, a process kill mid-video typically produces a playable file that OrphanRecovery restores cleanly; `MediaRecorder` (voice) has no equivalent resilience, so torn `.m4a`s are the common failure path. The order-log + existence filter ensures restored queues match the original capture order.

## Module layout

Package root: `com.example.bundlecam`. All sources under `app/src/main/java/com/example/bundlecam/`.

- `ui/capture/` — capture screen + VM + all capture-side UI. `CaptureScreen` composes the top-bar `ModalityPill`, the viewfinder `Box` with `Modifier.draggable` for the modality swipe carousel (translationX drives both the viewfinder and the control-row Row as "one slab" — see `ModalitySwipeMath` for the pure threshold math), `CameraPreview` (alpha=0 in VOICE so the camera stays bound but invisible), the conditional `VoiceOverlay` (scrolling amplitude bars via Canvas, fed by `VoiceController.amplitudeFlow`), and modality-aware control rows. Shutter is one of three siblings picked by `when (modality)`: `ShutterButton` (photo), `VideoShutterButton` (42dp→28dp morph dot→stop-square), `VoiceShutterButton` (58dp→28dp mic-in-white → stop-square). Gestures (`QueueStrip`, `QueueThumbnail`), divider arithmetic (`DividerOps`), first-run gesture onboarding (`GestureTutorial` with `seenTutorialSteps: Set<String>` for per-step persistence). Discard-undo window uses `TimedSlot<StagingSession>` from `ui/common/`. **UI is locked to portrait** (`screenOrientation=portrait` in the manifest) — icons counter-rotate via `Modifier.rotate(-deviceOrientation)` with shortest-arc animation.
- `ui/setup/`, `ui/settings/`, `ui/common/`, `ui/theme/` — first-run folder picker, settings (now includes Camera Mode: ZSL / EXT as a live-applied projection from DataStore), shared banner/picker, Material 3 theme with dynamic color on Android 12+.
- `data/camera/` — `CaptureController` owns `ProcessCameraProvider` + `ExtensionsManager` + `VideoCapture<Recorder>`, binds `Preview + ImageCapture + VideoCapture` concurrently under a `bindMutex` (try/catch falls back to photo-only if the device rejects the triple-bind — `combinedBindSupported` flag). `OrientationEventListener` snaps device degrees to cardinal and writes both `ImageCapture.targetRotation` and `VideoCapture.targetRotation` — the latter is frozen at `Recorder.start()` automatically, so mid-record writes are harmless no-ops. `takePicture()` (photo), `startVideoRecording(file)` / `stopVideoRecording(): RecordingResult` (video, `withAudioEnabled` omitted so mic stays free for voice), `setFlashMode(FlashMode.Off/Auto/On)` mutates live `ImageCapture.flashMode`. `bind()` re-applies `currentFlashMode` *after* installing the new capture. `ThumbnailDecoder` dispatches on file extension: jpg → BitmapFactory, mp4 → `MediaMetadataRetriever` (file or SAF URI), m4a → hand-drawn mic-glyph bitmap via Canvas. `isMediaFileReadable` is OrphanRecovery's torn-file probe (works for both mp4 and m4a).
- `data/audio/` — `VoiceController` wraps `MediaRecorder` (AAC-LC .m4a / 48kHz mono / 96kbps). `startRecording` returns a sealed `VoiceRecordingResult` (Success / Error / PermissionDenied / NotRecording); `stopRecording` returns the file with duration from `SystemClock.elapsedRealtime` deltas. Internal `amplitudeScope` polls `getMaxAmplitude()` at ~30Hz while recording, exposing `amplitudeFlow: StateFlow<Int>` for the overlay's waveform.
- `data/exif/` — `ExifWriter.stamp` (capture-time) and `stampFinalMetadata` (worker-time, single open/save). `OrientationCodec` converts between device degrees / `Surface.ROTATION_*` / ExifInterface tags. **Photos only** — videos use MP4 container rotation (`tkhd.matrix`) written by `Recorder`, and voice has no orientation semantics.
- `data/location/` — `LocationProvider` with 30s TTL cache, mutex-guarded, returns null on permission denial.
- `data/settings/` — `SettingsRepository` backed by DataStore Preferences; `SettingsState { rootUri, stitchQuality, shutterSoundOn, saveIndividualPhotos, saveStitchedImage, deleteDelaySeconds, deleteConfirmEnabled, seenTutorialSteps: Set<String>, cameraMode: CameraMode }` as a distinct-until-changed `Flow`. `cameraMode` (ZSL/EXT) was demoted from the top-bar EXT/ZSL toggle to a Settings row — **live-applied**, never frozen into the manifest. `seenTutorialSteps` replaced the old Boolean flag: the filter `TutorialStep.entries - seenTutorialSteps` determines which steps to show, so adding new tutorial steps in future releases doesn't re-trigger previously-dismissed ones. Upgrade path: on first read, if the legacy `seen_gesture_tutorial` Boolean is true, seed the set with the v1 step IDs (`commit, discard, deleteOne, reorder, divide`).
- `data/storage/` — `StagingStore` (internal FS, with `.order` append-log + `.discarded` marker + `allocateVideoOutput` / `allocateVoiceOutput` path returners), `SafStorage` (writes via `DocumentFile`, MIME by extension via `StorageLayout.mimeFor`, batches `listFiles()`, overwrites on name collision; `findOrCreateDir` re-`findFile`s after null `createDirectory` to survive concurrent-creator races), `StorageLayout` (canonical naming for all three modalities — `%03d` index width — **change here, not at call sites**), `BundleCounterStore` (per-date monotonic counter), `BundleLibrary` (SAF read-back, per-modality thumbnail URI selection, plus `listBundleFiles(bundle): List<BundleFile>` for LocalSend), `BundleFile` (`uri / fileName / size / mimeType / subfolder` — `subfolder` carries `"photos"|"videos"|"audio"|"stitched"|""` so the LocalSend uploader composes wire paths the receiver can mirror back into the bundle's directory layout).
- `pipeline/` — `PendingBundle` (kotlinx-serializable manifest with `version: Int = 2`, sealed `PendingItem { PendingPhoto, PendingVideo, PendingVoice }`), `ManifestStore` (v1-vs-v2 branched decoder, `encodeDefaults=true`), `WorkScheduler`, `BundleWorker` (+ pure `planRouting(manifest): RoutingPlan` companion for type-partitioned global-index assignment, testable via `BundleWorkerRoutingTest`), `Stitcher` (photo-only, layout math is a pure `companion` function `computeLayout` so tests don't need Bitmap allocation), `OrphanRecovery` (sealed `RestoredItem { Photo, Video, Voice }` with per-modality thumbnail construction).
- `network/localsend/` — clean-room **sender-only** implementation of LocalSend protocol v2.1. `LocalSendController` is the top-level coordinator (one base OkHttpClient, lazy-cached `Info` block, single discovery instance). `LocalSendDiscovery` runs the multicast send/listen on `224.0.0.167:53317` — `startUp` is `withContext(Dispatchers.IO)` so blocking socket calls don't trip `NetworkOnMainThreadException`; the receive loop checks `currentCoroutineContext().isActive` (NOT `scope.isActive`) so `cancelAndJoin` actually exits the loop. `LocalSendUploader.send` ships **all selected bundles in one prepare-upload session** (LocalSend's session is bound to the receiver's UI dismissal, so back-to-back sessions to the same peer would 409 BLOCKED). Per-peer SSL config layered on via `baseClient.newBuilder().sslSocketFactory(...)`; `FingerprintPinningTrustManager` extends `X509ExtendedTrustManager` directly (NOT just `X509TrustManager`, so the JDK doesn't wrap us with hostname-check that fails on our per-install cert's CN) and verifies the leaf cert's SHA-256 *during* the TLS handshake — match → return → session authenticates → `peerCertificates` populates. Files stream via a custom `RequestBody.writeTo(sink)` that pulls from `contentResolver.openInputStream(uri)` in 64 KB chunks, polls the calling Job's `isActive` between chunks for fast cancellation. Pure companions: `LocalSendUploader.wireNameFor`, `buildPrepareUploadRequest`, `LocalSendDiscovery.parseAnnounce` / `announceToPeer`, `LocalSendCertManager.computeFingerprintHex`. `LocalSendDtos.kt` mirrors every wire shape with `@Serializable` (Json: `encodeDefaults = true`, `ignoreUnknownKeys = true`, `explicitNulls = false`).
- `di/AppContainer.kt` — plain singleton, no DI framework. Constructed in `ReconApp.onCreate`. Holds `captureController`, `voiceController`, all storage/pipeline singletons, and the LocalSend stack (`localSendCertManager`, `localSendController`). `configureRoot(uri)` returns a `Job` launched on a public `appScope` (`SupervisorJob + Main.immediate`) — **do not call it from a composition-scoped `launch`**, or popping the screen mid-setup would cancel `ensureBundleFolders`. `appScope` is also what `LocalSendSheet.closeAndDismiss` uses to run `stopDiscovery` after the sheet's own scope has been cancelled (the sheet's `rememberCoroutineScope` is gone the instant the parent removes it from composition).

`ReconApp` implements `Configuration.Provider` for WorkManager; the manifest `<provider tools:node="remove">` block **disables WorkManager's default auto-init** so our provider is actually used (otherwise the default config wins the race against `Application.onCreate`).

## Conventions worth knowing

- **Log tag format**: `"Recon/<ClassName>"` as a private file constant.
- **Error surfacing**: recoverable failures write `state.lastError` for the banner; unrecoverable shouldn't happen (or they'd crash). Don't add silent `catch` blocks.
- **Commits across phases are atomic from the worker's view**: in `onCommitBundle`, save *all* manifests before enqueuing *any* worker — a mid-loop crash leaves no enqueued workers on partial state.
- **Gesture zones are siblings in a `Box`, not arbitrated**: the commit/discard `EdgeZone`s stack on top of the tray and `systemGestureExclusion()` goes flush to the screen edge. Don't try to merge them into a single gesture handler.
- **`EdgeZone` width ≠ destination-fill width**: edge zones grow on short queues (slack-based formula, capped at 33% per side) to catch swipes from further inward, but the tick/cross destination fill stays pinned at `EdgeZoneMinWidth` (60dp) anchored to the screen edge. Keep these two concerns decoupled when touching `QueueStrip` / `EdgeZone`.
- **Error banner overlays the queue strip** (peer of `UndoToast` / `BundleSavedShimmer` inside the queue `Box`), not a row above ZoomControl. Don't move it back inline — that pushes the shutter + queue down when an error fires. `ActionBanner` caps text at `maxLines = 2` with ellipsis so it stays within the 72dp queue-strip height.
- **`GestureTutorial` is a top-level scrim**, declared last inside `CaptureScreenContent`'s root `Box` so it draws above status/nav bars and the `DeleteGlow`. It consumes every pointer event (the `awaitEachGesture` loop eats unconsumed changes) — don't poke holes in the scrim, or users can accidentally swipe the (empty) queue underneath. The demo strip inside the overlay matches the real queue's 72dp height and is pinned at the bottom with a 12dp nav-bar gap so muscle memory transfers when the overlay dismisses.
- **CameraPreview pointer-input doesn't consume the down.** Compose's stock `detectTapGestures` calls `down.consume()` immediately, which starves the ancestor `Modifier.draggable` of the modality-swipe gesture — so `CameraPreview` rolls its own `awaitEachGesture { awaitFirstDown() ; waitForUpOrCancellation() }` for tap-to-focus and a matching multi-touch block for pinch-to-zoom. Don't replace these with `detectTapGestures` / `detectTransformGestures` — the swipe carousel will stop working.
- **Slab-swipe spring-back animation runs in `swipeScope` (rememberCoroutineScope), NOT the `onDragStopped`'s own scope.** `Modifier.draggable` rebuilds its internal pointer-input coroutine whenever the `onDragStopped` lambda reference changes — and `vm.setModality(target)` triggers exactly that, because the lambda captures `modality`. Running `slabOffset.animateTo(...)` inside the draggable scope gets cancelled mid-flight and the slab freezes partway. Keep the animation in `swipeScope`.
- **Per-modality control visibility is explicit.** Flash is PHOTO-only, lens-flip is PHOTO/VIDEO-only, ZoomControl is rendered with `alpha=0` in VOICE to preserve layout height (prevents the shutter row from jumping when switching modalities). CameraPreview is also `alpha=0` in VOICE (still composed so the camera stays bound — flipping alpha, not removing from tree). The `ModalityPill` disables every segment when `busy == Recording` but keeps the currently-selected one visually distinguishable (dimmed white background + dark text).
